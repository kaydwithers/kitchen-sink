Layout

The layout system is a powerful, flexible, highly advanced evolution of the traditional grid system.

It is based on inline-blocks. That means three things: 1. We don't have to use floats, offsets and clearfixes. Yay! 2. We can easily flip the layout, center it by just adding one class. 3. Since we use inline-block we unfortunately can't have spaces between the columns. We need to remove white space by inserting an empty comment between layout items. Refer to code examples below for more information.

  • .u-pt - padding-top on the object
  • .u-pr - padding-right on the object
  • .u-pb - padding-bottom on the object
  • .u-pl - padding-left on the object

Build it by adding classes with the CSS helpers that you have available. If something is too complex to build with helpers then custom CSS is acceptable.

A lot of our helper classes also work at different breakpoints. Try adding @s, @m, @l for responsive helpers. i.e ".u-fl@s"

ITCSS

ITCSS or Inverted Triangle CSS is a Scalable and Maintainable CSS Architecture and Methodology for large projects. It is not a framework but a collection of principles and metrics by which developers should group and order their CSS in order to keep it scalable, elegant, logical, and manageable.

We also use BEMIT (BEM + ITCSS = BEMIT) and prefix our classes in the codebase with a certain string in order to explain to developers what kind of job it does. This naming allows us to ascertain exactly what kind of job a class might have, how and where we might be able to reuse it (if at all), whether or not we can modify it, and much more.

Objects

o-

Signify that something is an Object, and that it may be used in any number of unrelated contexts to the one you can currently see it in. Making modifications to these types of class could potentially have knock-on effects in a lot of other unrelated places. Tread carefully.

Components

c-

Signify that something is a Component. This is a concrete, implementation-specific piece of UI. All of the changes you make to its styles should be detectable in the context you're currently looking at. Modifying these styles should be safe and have no side effects.

Utilities/Trumps

u-

Signify that this class is a Utility class. It has a very specific role (often providing only one declaration) and should not be bound onto or changed. It can be reused and is not tied to any specific piece of UI.

Hacks

_

Signify that this class is the worst of the worst-a hack! Sometimes, although incredibly rarely, we need to add a class in our markup in order to force something to work. If we do this, we need to let others know that this class is less than ideal, and hopefully temporary (i.e. do not bind onto this).

Hacks should be placed in the _shame.scss file.

If you have to do a hack, make sure you write an apology comment explaining why you had to do the hack. Include your name and the date. This helps us know why it was done and if it can be fixed at a later date.
